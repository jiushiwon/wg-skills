# 设计技能集成指南

> 本参考文档定义 `uniapp-app-genernate-skill` 在开发小程序页面与组件时，应如何调用 `ponytail`、`ui-ux-pro-max`、`frontend-design` 等技能，以兼顾代码精简与界面质量。

## 调用原则

在实现任何页面或组件前，先按以下顺序决策：

1. **是否需要设计感较强的界面？** → 先调用 `frontend-design` 或 `ui-ux-pro-max` 产出设计方案。
2. **实现方案是否可能过度设计？** → 调用 `ponytail` 审视，取最简可行实现。
3. **最终代码**：在美观与精简之间取得平衡，避免为"未来可能的需求"预留抽象。

## ponytail

### 何时调用

在写业务逻辑、组件、工具函数时，尤其是出现以下情况：

- 想新增一个抽象类/接口/配置，但当前只有一个实现；
- 想用第三方库实现一个几行代码就能搞定的事；
- 想为某个功能写大量注释或文档；
- 想提前做"以后可能用得上"的扩展。

### 使用方式

```text
/ponytail full
帮我以最小代码实现一个 uni-app 的倒计时按钮，要求点击后开始 60 秒倒计时，期间禁用。
```

### 在 uni-app 中的典型应用

- 能用 `uni.xxx` 标准 API 的，不用封装；
- 能用 CSS 实现的动画，不用 JS；
- 列表渲染用 `v-for`，不要额外套一层无意义容器；
- 状态管理只在跨组件共享时使用，单页面数据用 `ref`/`reactive`；
- 删除所有未使用的导入、变量、组件。

## ui-ux-pro-max

### 何时调用

需要系统级 UI/UX 设计决策时：

- 确定整体配色、字体、间距系统；
- 设计复杂页面（首页、数据看板、个人中心）的信息架构；
- 需要多风格对比（清新/极简/活泼/商务）；
- 需要检查 accessibility、响应式、视觉层次。

### 使用方式

```text
/ ui-ux-pro-max
为一款饮水打卡小程序设计首页，要求：清新健康风，主色 #10b981，包含今日饮水进度、快捷记录按钮、历史记录入口。输出 uni-app 页面结构建议和 SCSS Token 使用方式。
```

### 在 uni-app 中的典型应用

- 根据项目风格输出页面组件划分；
- 提供颜色、间距、圆角、阴影的具体 Token 建议；
- 审核已写好的页面，指出视觉层次和可用性问题；
- 为空状态、加载态、错误态提供设计建议。

## frontend-design

### 何时调用

需要高质量、具有辨识度的界面设计时：

- 首页、引导页、营销页、个人主页等关键页面；
- 需要避免"AI 味"、做出差异化的界面；
- 需要具体的 HTML/Vue/uni-app 代码实现建议。

### 使用方式

```text
/ frontend-design
为一款运动打卡小程序设计首页，要求：现代、有活力、避免模板感。输出 uni-app 模板代码，使用 SCSS 变量，不使用 emoji。
```

### 在 uni-app 中的典型应用

- 生成具体页面模板代码；
- 提供卡片、按钮、列表、图表等组件的视觉方案；
- 与主题系统结合，输出可直接落地的 Vue 代码。

## 推荐组合工作流

```
拿到页面需求
  → 调用 ui-ux-pro-max 确定信息架构与风格
  → 调用 frontend-design 生成页面模板
  → 调用 ponytail 审查并精简代码
  → 用 icon-forge 生成图标
  → 用 Pexels 工具获取真实图片
  → 写入项目并跑 lint
```

## 注意事项

- 不要同时调用多个设计技能做同一件事，避免建议冲突。
- `ponytail` 是"最后把关"角色，不是不要设计，而是设计后要剔除多余代码。
- 所有设计输出必须符合项目主题系统：颜色、间距、字体必须映射到已有 Token。
- 禁止在设计输出中使用 emoji 或渐变色/色块替代真实图片。
