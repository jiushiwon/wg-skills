# xhs-style-writer-skill - 小红书个人风格写作

> 通用爆款公式人人都在用，用出来就是 AI 味 2.0。真正带流量的是“像这个人写的”。

## 解决什么问题

- AI 写的小红书笔记一眼假，平台不推流。
- 同一个 prompt 出来的内容千篇一律。
- 想建立自己账号的固定人设，但每次都靠灵感。

这个 Skill 先帮你建一份可复用的**风格档案**，之后每次写笔记都按档案执行，最后再走一遍去 AI 味自检。

## 三种用法

### 1. 建立风格档案（第一次用）

贴 3-10 篇你的笔记原文：

```
这是我的 5 篇笔记：<粘贴原文>，帮我建风格档案
```

产出一份 `xhs-style-<账号名>.md`。样本越多，档案越准；少于 3 篇会明确提示“档案会偏”。

### 2. 按档案写笔记

```
按我的风格档案，写一篇“小基数减肥”的笔记
```

产出流程：选题 3 选 1 → 3 个不同套路的标题候选 → 正文 → 去 AI 味自检说明 → 5-8 个标签 → 首图文案建议。

### 3. 给现成文案去 AI 味

```
这篇文案帮我改得像人写的：<粘贴文案>
```

产出：改写稿 + 3-5 条“改前 → 改后”修改说明。

## 快速开始

1. 把本目录复制到 `~/.claude/skills/xhs-style-writer-skill/`（个人级）或项目 `.claude/skills/` 下。
2. 对 Claude 说任意触发语句即可，例如：
   - “写小红书笔记”
   - “帮我建风格档案”
   - “给这段文案去 AI 味”
3. 第一次用建议先建档，再写作。

## 设计要点

- **档案优先，公式兜底**：所有产出先看风格档案，公式只在样本不足时启用。
- **每条风格结论必须带原文例句**：无法验证的特征不进档案。
- **去 AI 味自检强制命中**：每篇至少命中 3 条，否则主动再改。
- **红线保护**：不编造、不冒充、不承诺流量；商业推广须标注。

## 常见问题

**Q：可以用对标账号的笔记建档吗？**
A：可以，但须确保你有权使用或已取得授权。禁止冒充他人、洗稿。

**Q：建档需要多少篇笔记？**
A：3-10 篇。≥5 篇置信度高；<3 篇会明确提示档案偏薄。

**Q：去 AI 味和 `ai-speech-detector` 有什么区别？**
A：本 Skill 专攻小红书文案的“人味”改写；如果只是检测/去除一段通用文字的 AI 味，用 `ai-speech-detector` 更合适。

**Q：能保证爆款吗？**
A：不能。只保证产出像本人、无 AI 味、合规。
