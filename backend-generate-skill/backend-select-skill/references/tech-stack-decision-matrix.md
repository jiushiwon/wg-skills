# 技术栈选型决策矩阵

本文件用于在生成后端项目时推荐合适的技术栈。决策应综合考虑项目类型、团队熟悉度、性能需求和运维复杂度。

## 决策维度

| 维度 | 说明 |
|------|------|
| 项目类型 | 企业级业务系统、快速 MVP、高并发服务、AI/数据型应用 |
| 团队背景 | 团队最熟悉的语言与框架 |
| 性能要求 | 并发量、延迟要求、资源占用 |
| 生态需求 | 第三方库、中间件、云原生支持 |
| 部署环境 | Docker / K8s / Serverless / 传统服务器 |

## 默认推荐

| 项目类型 | 推荐语言 + 框架 | 默认数据库 | 适用场景 |
|----------|-----------------|------------|----------|
| 企业级 / 复杂业务 | Java + Spring Boot | PostgreSQL | 大型系统、多模块、长生命周期 |
| 快速 API / AI / 数据 | Python + FastAPI | PostgreSQL | 快速迭代、异步 IO、机器学习 |
| 全栈 JS / 快速 MVP | Node.js + Express | PostgreSQL | 轻量、快速启动 |
| 全栈 JS / 企业风格 | Node.js + NestJS | PostgreSQL | 前后端同语言、TypeScript、模块化 |
| 高并发 / 微服务 | Go + Gin | PostgreSQL | 云原生、高性能、小体积 |
| 轻量 / Serverless | Node.js + Express 或 Python + FastAPI | PostgreSQL | 函数计算、简单服务 |

## 数据库选型

| 数据特征 | 推荐 | 说明 |
|----------|------|------|
| 强关系、事务 | PostgreSQL | 默认首选，功能全面 |
| 关系型但团队熟悉 MySQL | MySQL 8 | 兼容现有团队 |
| 灵活模式、文档型 | MongoDB | 原型快、非结构化数据 |
| 时序 / 日志 | InfluxDB / TimescaleDB | 仅在用户明确需要时推荐 |

## 与用户确认清单

1. 项目主要类型是什么？
2. 团队最熟悉哪种后端语言？
3. 是否需要强事务和复杂查询？
4. 预期并发量如何？
5. 是否有指定的数据库或中间件？
