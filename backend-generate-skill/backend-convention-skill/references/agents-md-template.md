# AGENTS.md 模板

本文件用于为新生成的后端项目创建 AGENTS.md。作为完整规范入口，CLAUDE.md 仅作简要重定向。

```markdown
# {{PROJECT_NAME}} 完整规范

## 1. 项目概述

- 产品名称：{{PROJECT_NAME}}
- 目标用户：
- 核心价值：
- 技术栈：{{STACK}}
- 数据库：{{DATABASE}}
- 第三方组件：{{THIRD_PARTY}}

## 2. 技术选型理由

- 语言框架：
- 数据库：
- 中间件：

## 3. 版本锁定

参见项目根目录 `versions.md`，其中记录运行时、框架、数据库、中间件的版本选择及原因。

## 4. 目录结构

```
{{DIRECTORY_TREE}}
```

## 5. 统一响应格式

所有接口返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

- `code >= 0`：成功
- `code < 0`：失败
- HTTP 状态码统一 200

## 6. 错误码定义

错误码闭集见项目根目录 `api-contract.md` 的"错误码汇总"节（其唯一来源为 `backend-convention-skill/references/response-format.md`）。本文件不另列码表；新增/调整错误码必须先改 `response-format.md` 并同步契约，禁止在此单独增删。

## 7. 数据库约定

- 表/集合前缀：`{{DB_PREFIX}}_`
- 字段命名：snake_case
- 主键：自增整数或 UUID
- 时间字段：`created_at`、`updated_at`

## 8. 环境变量

参见 `.env.example`。

## 9. 接口契约

参见项目根目录 `api-contract.md`。该文档是前后端对接的**唯一事实来源**，前端必须严格对齐：

- 路径、方法、Content-Type 与契约完全一致；
- 请求参数（路径/查询/body）的字段名、类型、必填与契约一致；
- 枚举值来自契约"枚举字典"节，不得新增、改写；
- 响应解析读取 `code` 判断业务状态，禁止使用 HTTP 状态码；
- 已处理契约列出的所有错误码，未列出的按 `-2000` 兜底。

**实现与契约冲突时，以契约为准**：后端修改实现，前端不得绕过。接口变更必须先更新契约，再改实现，最后通知前端。

## 10. 开发流程

1. 修改代码前先阅读 `api-contract.md`。
2. 新增接口必须**先更新** `api-contract.md`，再实现，最后通知前端。
3. 删除或修改字段类型、枚举含义属于破坏性变更，必须升级契约主版本号并通知前端。
4. 变更数据库前创建迁移文件。
5. 提交前运行 lint/build/test，并确保实现与契约一致。

## 11. 安全与部署

- `.env` 不提交。
- 生产环境不返回堆栈信息。
- 敏感操作记录审计日志。
```
