# 领域适配检测规则

本文档定义不同领域的特色 AI 模式检测规则。

---

## 一、学术论文

### 检测模式

| 模式 | 风险等级 | 示例 |
|------|----------|------|
| "The implications are significant" | 🔴 Critical | 直接陈述具体影响 |
| "highlighting the need for" | 🔴 Critical | 删除或改为"requires" |
| "It is worth noting that" | 🔴 Critical | 删除 |
| "In this study, we" | 🟠 High | 改为"We" |
| "Taken together, these results" | 🔴 Critical | 删除 |
| "These findings suggest" | 🟠 High | 直接陈述结论 |
| "Further research is needed" | 🟠 High | 具体说明需要什么研究 |
| "To the best of our knowledge" | 🟠 High | 删除 |

### 结构模式

| 模式 | 说明 |
|------|------|
| 摘要三段式 | 背景-方法-结论的机械结构 |
| "Introduction" 中的文献堆砌 | 连续引用而无观点 |
| "Discussion" 中的过度谦虚 | "may", "might", "possibly" 过度使用 |

## 六、领域识别规则

自动识别文本所属领域：

| 领域 | 触发条件 |
|------|----------|
| 学术论文 | 包含 "摘要"、"关键词"、"参考文献"、"methodology" 等 |
| 社媒/短视频 | 包含 "家人们"、"绝绝子"、emoji 密度 > 20% |
| 电商文案 | 包含价格、"抢购"、"限时"、"秒杀" 等 |
| 技术博客 | 包含代码块、"教程"、"干货"、技术术语密度高 |
| 公文/汇报 | 包含 "贯彻"、"落实"、"推进"、"加强" 等动词堆砌 |

识别到领域后，应用对应的特色检测规则。多领域混合时，分别列出各领域的检测结果。
