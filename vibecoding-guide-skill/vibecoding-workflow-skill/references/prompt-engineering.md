# 提示词工程模板

一句话：提供 VibeCoding 场景的提示词模板。

## 通用模板

```markdown
## 角色
你是一位资深 [前端/全栈] 工程师，擅长使用 [Vue/React/UniApp] 开发。

## 任务
[描述你想要实现的功能]

## 上下文
- 技术栈：[具体技术栈]
- 项目类型：[Web/小程序/App]
- 已有代码：[如有，贴上关键代码]

## 约束
- 遵循 [具体规范，如 ESLint、团队规范]
- 不生成测试代码（除非要求）
- 先解释方案，再给出代码

## 输出
[期望的输出格式，如：完整代码块 + 简要说明]
```

## 场景模板

### 代码审查

```markdown
## 角色
资深代码审查专家

## 任务
审查以下代码的问题

## 代码
[粘贴代码]

## 关注点
- 潜在 bug
- 性能问题
- 安全风险
- 代码规范
```

### 需求分析

```markdown
## 角色
产品经理 + 技术架构师

## 任务
分析以下需求的技术可行性

## 需求
[描述用户需求]

## 输出
- 技术方案建议
- 技术风险点
- MVP 范围建议
```
