# 10 · 常见问题（FAQ）

## 通用

### Q1: foundry 和其他 SKILL 是什么关系？
A: foundry **吸收**了它们的优点，**不替代**：
- `impeccable` → 设计法则（OKLCH、AI slop 测试）
- `ui-ux-pro-max` → 161 规则库
- `ui-audit` → 3-pillar 决策框架
- `frontend-code-doctor` → 代码审查
- `awesome-design-md` → 58 品牌 + 8 能力域

详细对比见 `D:\frontend-skills\` 各 SKILL 目录。

### Q2: 怎么调用 foundry？
A: 三种方式：
```bash
# 1. Skill 工具
Skill → "frontend-ui-foundry" → 子命令

# 2. 直接读 SKILL.md（Claude 自动路由）
读 SKILL.md → 输入子命令 → 执行

# 3. 单独跑脚本
node scripts/detect-stack.mjs
node scripts/extract-tokens.mjs
```

### Q3: 调色板 / 场景 / 品牌太多怎么选？
A: 用决策表：
- **调色板**：见 [06-调色板选择.md](06-调色板选择.md)
- **场景** → 品牌映射：见 [07-场景与品牌映射.md](07-场景与品牌映射.md)
- **品牌**：先看 8 场景默认品牌，需要时 `--brand` 覆盖

## Forge 相关

### Q4: forge 会修改我现有的代码吗？
A: **不会**。forge 是 greenfield（从零创建），用于新项目。
已有项目用 `optimize`。

### Q5: forge 输出的是什么？
A: 
- `tokens.css` / `tailwind.config.ts` 完整 Token
- 基础组件（Button、Input、Card）
- 页面骨架
- `forge-report.md` 决策报告

### Q6: --prose-only 是什么？
A: 只输出文案 + 页面结构 + 组件清单，不写实际代码。
适合：先评审方案，满意后再写实现。

## Optimize 相关

### Q7: optimize 会破坏我的业务代码吗？
A: **不会**。严格只改视觉层（颜色/字体/间距/圆角/阴影/动效）。
不改：API、数据结构、状态、路由、props、event、业务逻辑。

### Q8: 3 个策略怎么选？
A:
- **conservative**（低风险）：只统一不一致项
- **gradual**（中风险，推荐）：对齐到匹配品牌 80%
- **reshape**（高风险）：完全重塑

### Q9: optimize 失败怎么回滚？
A: 
```bash
git checkout HEAD -- .
```
所有 modify 在 git 工作区，可用 git 完整回滚。

## Audit 相关

### Q10: audit 评分怎么算？
A: 
```
总分 = pass * 1.0 + warn * 0.5 + fail * 0
满分 100
```

### Q11: audit 和 audit 子命令一样吗？
A: `audit` 是 foundry 的子命令，与 `impeccable` 的 `audit` 不同。
foundry 的 `audit` 输出更结构化（Markdown 报告 + 评分）。

## Tokens 相关

### Q12: 怎么导出为移动端？
A:
```bash
tokens export swift --output Theme.swift    # iOS
tokens export kotlin --output Theme.kt      # Android
```

### Q13: 调色板支持动态切换吗？
A: 支持。每套调色板都派生为暗色模式（专用 token，不直接反转）。
运行时可用 CSS 变量 / Tailwind class 切换。

## 设计相关

### Q14: 字号阶为什么是 1.25？
A: 1.25 是 5 度音阶，1.5 是 4 度，1.333 是 3 度。
1.25 提供最丰富的视觉层次（xs/sm/base/lg/xl/2xl/3xl/4xl/5xl 共 9 级），且不夸张。

### Q15: 4pt 网格为什么？
A: iOS / Material 通用标准，对齐到设备像素比整数倍，渲染清晰。
iOS 设计稿 1px = 1pt，4pt 是最小有意义间距单元。

### Q16: 触摸目标为什么 ≥44pt？
A: Apple HIG 规范，4 平方毫米的手指可点击区域。
低于此值会导致误触和 a11y 问题。

### Q17: prefers-reduced-motion 怎么实现？
A: 在全局 CSS 加：
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
模板见 `templates/tokens.css.tpl` 末尾。

## 故障排除

### Q18: scripts 跑不了？
A: 需要 Node 18+。`node --version` 检查。
脚本都是 `.mjs`（ES Module），无需 npm install。

### Q19: extract-tokens 没找到文件？
A: 默认扫描 `cwd`。指定路径：
```bash
node scripts/extract-tokens.mjs /path/to/project
```

### Q20: AI Slop 模式命中太多？
A: 常见原因：
- 用了 Inter / Roboto（替换为 Geist / General Sans）
- 用了 `bg-gradient-to-r` + 文字（改用纯色）
- 用了 9 个相同卡片（改用变化尺寸）

参考 [09-AI-Slop-黑名单.md](09-AI-Slop-黑名单.md)。

### Q21: 想添加新场景 / 调色板 / 品牌？
A:
1. 在 `references/scenarios/` 新建 `<name>.md`，按现有 8 个格式
2. 在 `references/tokens/color-palettes.md` 追加新调色板
3. 在 `references/brands/index.md` 添加品牌 + 在 `brands/` 新建详细卡
4. 跑 `bash demo/verify.sh` 验证

### Q22: 怎么贡献给其他项目？
A: 直接复制 `frontend-ui-foundry/` 目录到目标项目根。
- `references/` 提供知识
- `scripts/` 提供工具
- `templates/` 提供 Token 起点
- `SKILL.md` 作为入口
